Deploy Django 2 on Heroku

Shubham Aggarwal
Oct 19, 2018

I am using python 3.7 and visual studio code as my text editor. You will need a free heroku account, heroku-cli and git installed. we will create simple application and deploy it on heroku.

Step 1

Create a fresh new django project(myproject). you can name it anything do you want. After, create an app(myapp) in our project.

// terminal or command line
$ django-admin startproject myproject
$ cd myproject
$ python manage.py createapp myapp

Step 2

Now create a route in myproject/urls.pyand it’s handler in myapp/views.py

// myapp/views.py
from django.http import HttpResponse
def index(request):
    return HttpResponse('<p>Hello World!</p>')

Now add the handler with /route

// myproject/urls.py
from django.contrib import admin
from django.urls import path
from myapp.views import index

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', index)
]

Step 3

Now, Install some dependencies

$ pip install django-heroku gunicorn

django-heroku: official package by Heroku which made our deployment simple.
gunicorn: WSGI application server

Step 4

Now, In myproject/settings.py file add below content

# djang-heroku setting
import django_heroku
django_heroku.settings(locals())
# static files settings
STATIC_URL = '/static/'
# location where you will store your static files like bootstrap
STATICFILES_DIRS = [
   os.path.join(BASE_DIR, "static"),
]
# location where django collect all static files
STATIC_ROOT = os.path.join(BASE_DIR, "staticfiles")

Step 5

Now, create static directory where manage.py file exists. if you do not have any static file then add an empty file in thestatic directory let say .gitkeep 
because Git does not track empty directories otherwise add your static files in the staticdirectory.

Step 6

Now, run a command in your terminal. it will create a file requirements.txt and list all your dependencies.

pip freeze > requirements.txt

Step 7

Now, create a file runtime.txt in your root project and add your python version in it.

$ echo "python-3.7.0" >> runtime.txt

Step 8

Now, create a file Procfile without any extension in your root project directory.

$ echo "web: gunicorn <your_project_name>.wsgi" >> Procfile
Procfile is a mechanism for declaring what commands are run by your application’s dynos on the Heroku platform.

Step 9

Now commit your changes

$ git init
$ git add .
$ git status
$ git commit -m "init commit"

Step 10

One last step, push code to heroku

$ heroku login // enter your email and password
$ heroku create <your website name(let say "myfirstwebsite")>
$ git push heroku master 
$ heroku open // automatically open your website in your browser

Step 11

Just send your website link to your friends and enjoy 😎. I hope this helps you and don’t forget to clap 👏
